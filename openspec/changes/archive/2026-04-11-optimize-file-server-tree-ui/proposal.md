## Why

当前 `file-server` 已经具备文件树、上传、新建文件夹和删除能力，但主要操作仍分散在右侧面板和内容列表中，左侧树更多只是导航入口。这与文件管理场景中“以目录树为主操作面”的预期不一致，也让界面的主次关系不够稳定。

现在需要把 `file-server` 的交互中心收回到左侧树区域，让用户围绕目录树完成选中、上传、新建和删除，右侧内容区先退化为空白占位，为后续内容视图扩展留出清晰边界。

## What Changes

- 重构 `file-server` 页面布局，把左侧区域升级为主要操作面
- 将文件树从“仅导航”调整为“导航 + 操作”的复合工作区
- 让新建文件夹、上传文件、删除目标等主要动作围绕树节点进行，而不是依赖右侧内容面板
- 将右侧内容区改为空白占位态，明确本次不继续扩展详情视图或文件列表能力
- 保持现有后端接口不变，前端在现有文件服务 API 基础上完成交互重组

## Capabilities

### New Capabilities

- `file-server-tree-workspace`: 定义 `file-server` 以前端树工作区为中心的布局、交互入口和空内容区行为

### Modified Capabilities

- 无

## Impact

- 影响前端代码：`file-server/src/App.tsx` 及其相关样式拆分文件
- 影响前端交互：左侧树区域承担主要文件操作入口，右侧内容区暂为空白占位
- 影响用户体验：文件操作路径从“右侧面板主导”切换为“左侧树主导”
- 影响接口：无新增后端 API，无后端协议变更
