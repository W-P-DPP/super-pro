## Context

当前 `frontend-template` 已经通过 `GET /api/site-menu/getMenu` 获取目录数据，但 `src/data/tool-directory.ts` 中的 `toPublicIcon` 仍会把后端返回的 `icon` 字段截取文件名后改写为前端本地 `/site-icons/<fileName>`。这意味着目录区和内容区卡片区虽然使用了后端菜单数据，却没有真正使用后端返回的图标地址。

仓库现状还包括两个约束：
- 后端已经提供 `general-server/public` 静态资源服务，菜单图标可通过 `/icons/...` 直接访问。
- 前端站点头部品牌图标仍是本地站点资源，本次需求只覆盖目录区菜单图标和内容区卡片图标，不要求替换品牌图标。

## Goals / Non-Goals

**Goals:**

- 让目录区菜单图标与内容区卡片图标直接以 `siteMenu.icon` 作为来源渲染。
- 去掉前端基于文件名的 `/site-icons/*` 本地映射逻辑，避免前后端图标配置分裂。
- 为图标地址增加统一的前端规范化逻辑，仅处理空值和相对路径格式，不改变后端图标资源归属。
- 补充前端测试，验证图标解析与渲染契约。

**Non-Goals:**

- 本次不改动后端 `siteMenu` 接口结构。
- 本次不替换站点头部品牌图标、favicon 等非菜单图标资源。
- 本次不新增图标上传、图标库管理或图标编辑能力。

## Decisions

### 决策 1：用“路径规范化”替代“本地图标映射”

前端将删除现有 `toPublicIcon` 这类“提取文件名后重写到 `/site-icons/*`”的逻辑，改为统一的后端图标路径解析函数。解析规则如下：

- `icon` 为 `http://` 或 `https://` 时直接使用。
- `icon` 为 `/icons/tool.svg` 这类根路径时直接使用。
- `icon` 为 `icons/tool.svg` 这类相对路径时补成 `/icons/tool.svg`。
- `icon` 为空或占位值时，使用统一的后端静态资源兜底路径，例如 `/icons/tool.svg`。

选择该方案而不是继续维持文件名映射的原因：

- 保证图标来源明确属于后端菜单数据。
- 后端更新 `icon` 字段后，前端无需同步复制本地静态文件。
- 只保留最小必要的路径容错，不再让前端隐式决定实际图标文件。

### 决策 2：目录区与内容区共用同一套图标解析函数

`AppLayout.tsx` 的目录区菜单项和 `HomePage.tsx` 的分组标题、内容卡片都必须共用同一个图标解析工具函数，而不是各自拼接路径。这样可以保证：

- 目录区与卡片区的图标展示行为一致。
- 后续若后端 `icon` 规范调整，只需要修改一处适配逻辑。
- 单元测试可以集中覆盖图标规则，不需要分别为多个组件维护重复断言。

### 决策 3：前端仅对菜单与卡片图标去本地化，不清空全部 `site-icons` 目录

本次需求聚焦“目录区与内容区卡片图标”。站点品牌图标 `tools.png` 当前仍用于布局头部，不属于菜单接口返回范畴，因此设计上仅移除菜单和卡片对 `/site-icons/*` 的依赖，不要求一次性删除前端全部站点静态图标资源。

## Risks / Trade-offs

- [风险] 后端若返回空字符串或错误路径，前端图标可能加载失败
  -> Mitigation：保留统一中文语义的后端静态图标兜底路径，并对空值做规范化处理。

- [风险] 后端返回相对路径格式不统一，可能出现 `/icons/...` 与 `icons/...` 混用
  -> Mitigation：在前端适配层只做轻量路径标准化，不做文件名映射。

- [风险] 目录区和卡片区若仍存在隐藏的本地图标拼接，可能导致部分区域行为不一致
  -> Mitigation：集中梳理 `AppLayout.tsx` 和 `HomePage.tsx` 的全部菜单图标入口，并补充测试覆盖。

## Migration Plan

1. 重构 `src/data/tool-directory.ts` 中的图标解析方法，去掉 `/site-icons/*` 文件名映射。
2. 更新 `AppLayout.tsx` 和 `HomePage.tsx`，让目录区与内容区统一消费新的图标解析结果。
3. 补充或更新前端测试，验证后端图标地址、空值兜底与路径规范化行为。
4. 构建或执行前端测试，确认目录区和卡片区图标展示正常。

回滚策略：

- 若接入后发现线上图标字段质量不足，可回滚到旧的本地图标映射实现。
- 回滚只影响前端图标适配，不影响已接入的 `siteMenu` 数据源与后端接口。

## Open Questions

- 后端后续是否会约束 `icon` 字段必须始终返回根路径格式，例如统一为 `/icons/...`？如果会，本次前端路径规范化可以在后续继续收敛。
