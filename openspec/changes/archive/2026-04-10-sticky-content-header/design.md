## Context

当前 `AppLayout` 已有一个 `sticky top-0` 的顶栏，但其高度通过局部类名 `h-[3.75rem]` 写死，只作用在内容区；侧边栏头部则通过 `py-4` 和内部内容自然撑开，没有统一的显式高度契约。`HomePage` 的根容器和分组锚点使用了固定的 `scroll-mt-24`，这会在头部高度调整后产生错位风险。

仓库的前端规范要求：

- 继续使用现有 shadcn 组件体系和 token，不另起一套视觉语言。
- 通过清晰结构、边界、间距来表达层级，而不是额外装饰。
- 对吸顶区域、滚动偏移和响应式行为做显式设计，不能依赖默认行为。
- 浅色和深色主题都要维持一致的布局节奏。

## Goals / Non-Goals

**Goals:**

- 为侧边栏头部和内容区头部建立统一高度常量或样式契约。
- 保证内容区头部在主内容滚动时持续可见。
- 保证首页分组锚点滚动到位后，不会被吸顶头部遮挡。
- 保持现有交互、数据渲染和视觉语言不变。

**Non-Goals:**

- 不改动后端接口与数据结构。
- 不重做侧边栏信息层级和首页卡片样式。
- 不增加新的导航、筛选或搜索能力。

## Decisions

### 决策 1：用共享头部高度变量统一侧边栏头部与内容区头部

在 `AppLayout` 中引入统一的头部高度常量，并同时应用到：

- `SidebarHeader`
- 主内容 `<header>`

这样可以避免两侧顶部各自通过 padding 和内容高度“碰巧对齐”，后续若要微调头部节奏，只需要改一处。

### 决策 2：把内容区头部与页面滚动内容分层

内容区头部继续放在 `AppLayout` 中作为全局布局头部，主页滚动内容放在其下方独立区域。吸顶行为由布局层控制，而不是让页面内部再次套一层吸顶壳，避免双层 sticky 相互覆盖。

### 决策 3：用 CSS 变量驱动锚点偏移

将首页分组的 `scroll-mt-*` 固定值改成基于头部高度的变量偏移，例如：

- `scroll-margin-top = 头部高度 + 页面内容顶部间距`

这样在未来头部高度变化时，分组锚点仍然能稳定落在可见区域内。

## Risks / Trade-offs

- [风险] 移动端或窄屏下头部内容折行后可能撑高。
  -> Mitigation：保持头部内部为单行布局，必要时隐藏次级元素，而不是允许头部自增高度。

- [风险] sticky 层与页面内容叠放后可能出现背景穿透。
  -> Mitigation：保留半透明背景和 `backdrop-blur`，并确保 sticky 层有明确背景色。

- [风险] 锚点滚动仍可能受外层间距影响。
  -> Mitigation：统一从页面根容器和分组容器同时校正滚动偏移。

## Migration Plan

1. 在布局层新增共享头部高度常量与样式变量。
2. 调整 `AppLayout` 侧边栏头部和内容区头部高度。
3. 调整 `HomePage` 根容器、搜索区和分组锚点的滚动偏移。
4. 构建前端并检查浅色/深色主题下的布局表现。
