# 02 MVP 范围

## 范围内

### 1. 公开需求信号采集

- 支持手工导入和定时采集
- 将多种公开来源统一成一套 `raw_signal` 结构
- 保留来源、发布时间、批次和原始快照

### 2. 信号清洗

- 去除 HTML、样式和无关噪声
- 统一语言和基础字段
- 标记重复内容
- 做基础脱敏
- 过滤明显垃圾或广告内容

### 3. 问题聚类

- 以“用户问题相似度”聚类，而不是只按关键词聚合
- 输出聚类标题、摘要、代表信号、规模和置信度
- 给出不应合并的边界说明

### 4. 机会摘要与证据整理

- 从 `problem_cluster` 生成 `opportunity_brief`
- 汇总代表性证据、用户角色、问题陈述、边界说明和信息缺口
- 为后续评分和 PRD 生成提供统一输入对象

### 5. 可配置机会评分

- 评分维度由 `scoring_profile` 配置，不在数据模型中写死
- 默认建议维度包括：需求强度、战略匹配度、实现可行性、流量或分发潜力、结果可见性、证据置信度
- 商业化潜力或付费意愿只能作为可选维度，不能成为固定前提
- 输出总分、分项分数、评分理由、关键风险和待验证问题

### 6. PRD 草案生成

- 仅对通过 Gate 1 的机会生成结构化 `prd_draft`
- 至少包含背景、目标用户、问题定义、方案假设、范围、风险、开放问题和引用证据

### 7. 人工审查与显式门禁

- 审查者可以查看证据链、评分理由和 PRD 草案
- Gate 1 负责判断是否允许进入 PRD 生成
- Gate 2 负责判断 PRD 草案是否通过、退回修改或拒绝
- 所有门禁决定都进入审计记录

### 8. 状态流转与可审计性

- 每个核心对象都有明确状态
- 每次状态变化都记录操作者、时间、原因和关联执行记录
- 每次 AI 执行都记录提示词版本、模型版本和输入输出快照

## 范围外

- 登录态、付费态或私有数据源抓取
- 自动路线图排序和自动立项
- 自动拆解成研发任务、测试用例或发布计划
- 复杂协同编辑器、批注系统和 BI 大屏
- 对现有服务做侵入式重构或强耦合接入
- 用一期结果直接替代用户访谈、市场验证或产品判断

## 一期范围边界

- 数据边界：只处理公开文本信号和手工导入内容，不处理私域数据
- 结果边界：一期输出止于“已审查的 PRD 草案”，不进入研发执行
- 决策边界：AI 只提供候选结论，最终是否进入 PRD、是否通过 PRD 仍由人工决定
- 集成边界：文档先定义新对象、新状态和新 API，不要求立即接入现有业务系统
- 运营边界：只要求形成最小可运行闭环，不要求一期覆盖全部来源和全部业务线

## MVP 验收边界

- 一条信号可以从采集流转到 Gate 2 待审状态
- 一个 `problem_cluster` 可以产出一个可审查的 `opportunity_brief`
- 一个 `opportunity_brief` 可以基于指定 `scoring_profile` 产出一个 `opportunity`
- 一个通过 Gate 1 的机会可以生成一个 `prd_draft` 并完成一次 Gate 2 审查
- 任意输出都能追溯回原始证据、AI 执行记录和人工门禁决定

## 约束

- 正确性、可追溯性和门禁清晰度高于自动化率
- 重跑必须生成新版本或新执行记录，不能覆盖历史
- 评分模型必须可按策略配置调整，不能把单一商业假设写死
- 未来接入 super-pro 必须以增量方式完成，不破坏现有服务
