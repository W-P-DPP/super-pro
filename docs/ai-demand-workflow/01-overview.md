# 01 总览

## 目标

一期北极星是建立一条从公开需求信号到“可审查 PRD 草案”的稳定流水线。系统负责整理证据、生成候选结论和草案，不负责替代人做最终产品决策。

一期必须同时满足以下要求：

- 端到端可运行：公开信号可以进入完整流程并产生待审结果
- 全链路可追溯：任何结论都能回溯到源信号、提示词、模型和人工决策
- 关键节点可人工把关：PRD 生成前和 PRD 生成后都有显式门禁
- 结果可重放：重复执行会生成新版本或新执行记录，不覆盖历史

## 一期成功指标

建议把以下指标作为一期验收口径：

- `100%` 的 `prd_draft` 都有明确的 PRD 前门禁通过记录
- `100%` 的机会评分和 PRD 草案都能追溯到 `opportunity_brief` 与原始证据
- `>= 80%` 的成熟聚类能产出包含证据摘要、边界说明和缺口说明的 `opportunity_brief`
- `>= 70%` 的首版 `prd_draft` 被产品评为“可审查”，退回原因不应是证据链缺失

## 主要用户

- 研究或运营：查看原始信号、清洗结果、聚类结果和证据包
- 产品经理：审查机会评分、PRD 前门禁材料和 PRD 草案
- 团队负责人：关注吞吐、门禁通过率、退回原因和来源质量
- AI 或系统负责人：维护来源配置、评分策略、提示词版本和模型配置

## 端到端流程

1. 从论坛、社区、评论、问答或手工导入收集公开需求信号
2. 对原始内容做清洗、归一化、去重、脱敏和噪声过滤
3. 将相似用户问题聚成 `problem_cluster`
4. 基于聚类生成 `opportunity_brief`，整理证据、边界和信息缺口
5. 使用可配置的 `scoring_profile` 对 `opportunity_brief` 打分，生成 `opportunity`
6. 进入 Gate 1，决定是否允许生成 PRD
7. 仅对 Gate 1 通过的机会生成 `prd_draft`
8. 进入 Gate 2，由人工审查 PRD 草案并给出结论
9. 记录所有状态变化、AI 执行记录和人工决策

## 运营模型

### AI 角色

- 信号整理助手：把原始文本转成结构化、可用、已脱敏的信号
- 聚类与证据助手：形成问题聚类并产出 `opportunity_brief`
- 评分助手：按照 `scoring_profile` 输出分项分数、理由和不确定性
- PRD 起草助手：根据已通过门禁的机会和证据草拟结构化 PRD
- 审查辅助助手：汇总差异、证据缺口和建议修改项

### 人类角色

- 来源负责人：决定采集哪些公开来源，维护采集质量和合法性边界
- 研究或运营：抽查信号质量、聚类边界和证据完整性
- 产品经理：负责 Gate 1 和 Gate 2 的业务判断
- 团队负责人：在策略冲突、优先级冲突时做最终裁决
- AI 或系统负责人：维护评分策略、提示词、模型和失败重试规则

### 决策门禁

- Gate 1，PRD 生成前：输入为 `opportunity_brief`、评分结果和策略配置；输出只能是“允许生成 PRD / 退回补证据 / 直接阻断”
- Gate 2，PRD 生成后：输入为 `prd_draft`、引用证据和历史评论；输出只能是“通过 / 退回修改 / 拒绝归档”

## 核心输出

- 结构化信号仓
- `problem_cluster`
- `opportunity_brief`
- 可配置评分后的 `opportunity` 待办池
- `prd_draft`
- 完整审计轨迹和状态历史

## 非目标

- 抓取需要登录或私有权限的数据源
- 自动拍板路线图或替代产品负责人做优先级决策
- 自动拆解工程任务、自动写代码或自动发布
- 一期内深度改造现有 super-pro 服务和前端
- 一期内建设复杂 BI、协同编辑器或经营结果闭环平台
